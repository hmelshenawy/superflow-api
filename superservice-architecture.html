Job Lifecycle — State Machine

Every job must pass through defined states. Never skip states in code — always transition explicitly.

STATE 01
NEW
Job created, vehicle + customer attached
→
STATE 02
INSPECTION
Tech filling checklist + photos
→
STATE 03
PRICED
Advisor added labour/parts estimates
→
STATE 04
SENT
Customer approval link dispatched
→
STATE 05a
APPROVED
All items authorised by customer
→
STATE 05b
PARTIAL
Some items approved, some deferred
→
STATE 05c
DECLINED
Customer declined — deferred work created
→
STATE 06
COMPLETED
Authorised work done, finalised in DMS

02System Layers Detail

Each layer has a strict responsibility. No layer should reach past its neighbour.

📱 Flutter Mobile App

Technician-only. Works offline. Syncs on reconnect.

  • 🔍VIN scan / manual lookup
  • 📋Pull inspection template from API
  • ✏️Fill checklist items with urgency levels
  • 📷Capture photos/videos per item
  • 💾Store draft locally (Hive/SQLite)
  • ☁️Background sync when online
  • 🔔Push notifications for job assignments

🌐 React Web App (Advisor/Admin)

Always online. Full job management dashboard.

  • ➕Create jobs / import from DMS
  • 📊Review inspection findings live
  • 💰Add labour + parts pricing
  • 📤Send estimate to customer
  • ✅Track approval status in realtime
  • 📁Manage deferred work queue
  • ⚙️Admin: templates, rates, users, integrations

🔗 Customer Portal (React)

Separate app. Accessed via secure link — no login required.

  • 🔐Token-based access (no password)
  • 📋View recommended work items
  • 🖼️Watch photos/videos evidence
  • ✅Approve or decline per line item
  • 💬Optional comment/message
  • 📧Receive confirmation after decision

⚙️ NestJS API Backend

Modular monolith. All business logic lives here.

  • 🏗️Domain modules with clear boundaries
  • 🔄State machine transitions enforced server-side
  • 📡WebSocket gateway for advisor realtime updates
  • 🔁BullMQ workers for async tasks
  • 🗂️Event emitter for cross-module communication
  • 🔌Integration adapters isolated in own module

08Security Guidelines

Workshop systems handle customer PII and financial decisions. Non-negotiable rules from day one.

🔐 Authentication & Tokens

  • 🎟️JWT access tokens — 15 min expiry. Refresh tokens — 30 days, stored hashed in DB
  • 🔗Customer approval tokens — 72h expiry, SHA-256 hashed, single-use, IP logged
  • 🚫Revoke all refresh tokens on password change or logout-all action
  • 📱Mobile app stores refresh token in secure storage (Flutter Secure Storage), never plain SharedPreferences

🛡️ API Security

  • 🚧Rate limiting on all public routes — especially /auth/login and portal token routes
  • ✅Server-side validation on all inputs — never trust mobile client data
  • 🔒RBAC guard on every route — no route is unprotected by default
  • 🌐CORS locked to known origins — advisor app, customer portal, mobile
  • 🔍Separate API base paths: /api/v1/internal vs /api/v1/portal

📁 Media & File Security

  • 🔗Never expose raw S3 keys or paths to clients — always generate signed URLs server-side
  • ⏰Signed URLs expire in 1 hour — regenerated on demand
  • 🧹Scan uploads for malware before confirming — run async on worker
  • 📐File type validation on server — check MIME type, not just extension
  • 💾Max file size enforced both client-side and server-side

📋 Data & Compliance

  • 🔐Encrypt PII at rest — customer email, phone in encrypted columns
  • 📝Full audit log on all state changes, price edits, approvals
  • 🗑️Data deletion: anonymise customer data on request, keep job records for audit
  • 🔒MinIO bucket — private, no public read, all access via signed URLs
  • 🌐HTTPS enforced via Caddy — HTTP redirects to HTTPS automatically

Goal: A job can go from creation to customer approval end-to-end. No gaps in the main workflow.

P3

DMS integration VIN auto-lookup Parts pricing connector Advanced tax rules Alternate parts suggestion OEM-specific templates Analytics dashboard Approval rate reporting Deferred value tracking Automation rules engine Multi-workshop support

⚠️ Mistakes to Avoid

❌ Starting with microservices — use modular monolith first
❌ Hardcoding inspection forms — make templates configurable from day one
❌ Mixing technician and customer flows in one app
❌ Storing media as generic attachments — link to inspection items
❌ No state machine — transitions must be enforced server-side
❌ Synchronous DMS integration — always async with retry queue
❌ No audit trail — every price change and approval must be logged
❌ No deferred work model — declined ≠ forgotten
❌ Raw file URLs in API responses — always signed, always expiring
